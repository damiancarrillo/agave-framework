A collection of default Converter s that Agave uses to convert from {@code String} parameters to higher level objects. The {@code String} parameters can come from form inputs, request parameters, or URI pattern positional parameters, in that order where the latter provides greater specificity.

An example of each is:

Form Inputs
Form inputs come from a submitted HTML form, where the parameter name is the name supplied with some input, eg: 
  <form method="post" action="${contextPath}/some/uri">
    <input type="text" name="parameterNameToBeConverted" />
  </form>
Request Parameters
Request parameters come from the HTTP request query string, eg: 
  http://localhost:8080/contextPath/some/resource?parameterNameToBeConverted=parameterValue
URI Pattern Positional Parameters
URI pattern positional parameters are variables whose value are supplied with the request URL and are embedded withing the actual URL and not the query string. These positional params are specified by variable delimiters in the HandlesRequestsTo annotation. For example, the following annotation has a positional parameter: 
  @HandlesRequestsTo("/profile/${username}")
  public Destination displayProfile(HandlerContext) {
      ...
  }
